2. Important Concepts and Definitions


Graph

A Graph is a data structure made of:

Vertices (Nodes) → points
Edges → connections between nodes

Example:

0 ---- 1
|      |
2 ---- 3

Vertices = 0,1,2,3

Edges = connections between them.


BFS (Breadth First Search)

BFS visits nodes level by level.

It uses a Queue (FIFO).

Example:

Start = 0

Visit:
0 → 1 → 2 → 3

Applications:

Shortest path
Network traversal
Web crawling


DFS (Depth First Search)

DFS goes deep first then backtracks.

It uses a Stack (LIFO).

Example:

0 → 1 → 3 → 2

Applications:

Maze solving
Cycle detection
Path finding


Parallel Computing

Parallel computing means:

Multiple tasks run simultaneously using multiple CPU cores.



Instead of:

1 task at a time

We do:

Many tasks together

This increases speed.


OpenMP

OpenMP is a library used for parallel programming in C/C++.

It creates multiple threads automatically.


Thread

A Thread is a lightweight execution unit.

Example:

1 thread → normal execution
4 threads → 4 tasks together


Queue

Queue works on:

FIFO = First In First Out



Example:

1 2 3

Remove:
1 first

Used in BFS.


Stack

Stack works on:

LIFO = Last In First Out



Example:

1 2 3

Remove:
3 first

Used in DFS.


Adjacency List

Used to store graph efficiently.

Example:

0 → 1,2
1 → 0,3
2 → 0,3
3 → 1,2


Critical Section

A Critical Section allows only one thread at a time.

Used to avoid:

data corruption
race condition


Race Condition

When multiple threads modify same data simultaneously.

Example:
Two threads pushing into queue together.

Can cause wrong output/crash.
